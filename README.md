# The Sound of Bees 🐝
### Stop signal detection and automatic audio–video monitoring in beehives

This project implements an end-to-end multimodal framework for the automatic monitoring of honeybee behaviour inside the hive, focusing in particular on the **stop signal**, a short vibro-acoustic event associated with a head-butting interaction, described in the literature as a form of inhibitory feedback within waggle-dance communication.

The system continuously acquires raw audio and video data, processes them to detect acoustically plausible stop-signal candidates, constrains them in time and space, and finally returns synchronized audio–video clips centred on each candidate event. The goal is to reduce the need for prolonged manual observation, typically required by studies on this signal, and to provide a practical tool for building quantitative datasets.

## Academic context

This repository contains the code developed for my Master's thesis in **Music and Acoustic Engineering** at **Politecnico di Milano**, academic year 2025/2026, under the supervision of Prof. Fabio Antonacci and the co-supervision of Dr. Andreagiovanni Reina and Alberto Doimo.

The project was carried out entirely at the **CASCB (Centre for the Advanced Study of Collective Behaviour)**, University of Konstanz.

## System architecture

The framework was built around an observation hive, equipped with two Ximea cameras for video acquisition and an array of 32 MEMS microphones (16 per side), connected via 8 MCHStreamer audio interfaces. The entire system operates in complete darkness, with infrared illumination used for video recording to avoid disturbing the bees.

The workflow consists of three main stages:

- **Data acquisition**: synchronized multichannel recording across multiple devices via a JACK/ALSA-based architecture
- **Cloud transfer**: upload of raw data to the university cloud infrastructure for backup and remote processing
- **Post-processing**: extraction of acoustic candidates and generation of synchronized audio–video clips


## Detection approaches

Two alternative candidate-extraction pipelines were developed and compared:

- **DSP-based pipeline**: based on Harmonic-to-Noise Ratio (HNR), high precision but low recall — best suited as a conservative candidate-retrieval tool
- **Animal2vec-based pipeline**: transformer-based framework for rare-event bioacoustics, fine-tuned on a labelled stop-signal dataset

Both pipelines were evaluated on a labelled dataset and on current-season recordings, with detailed results and limitations discussed in the thesis.

## Repository structure

| Folder | Content |
|---|---|
| `data_acquisition/` | Code for real-time audio/video acquisition (MCHStreamer hardware, Ximea cameras) |
| `data_transfer/` | Scripts for transferring raw data to the cloud infrastructure (based on rclone/rsync) |
| `postprocessing/` | Acoustic analysis of raw recordings and generation of audio–video clips centred on acoustically plausible stop-signal candidates |
| `detection_through_animal2vec/` | Fork of the official [Animal2vec](https://github.com/livingingroups/animal2vec) repository, adapted with custom config files and inference scripts |
| `database_analysis/` | Scripts to analyze, validate, and plot results generated by the post-processing stage |
| `classes/` | Shared Python classes underlying most of the other scripts in the repository |
| `old_attempts/` | Approaches explored on central topics of the project (multi-device synchronization, precise source localization) that yielded only partially positive results and were not integrated into the final workflow |
